# State-vector benchmark: FastSim vs cuStateVec

This directory provides scripts to benchmark FastSim against NVIDIA cuQuantum **cuStateVec** on the same workload (qubit count, gate sequence, and environment). Metrics include wall-clock time for one full circuit forward and optional GPU memory.

## Requirements

- **FastSim**: PyTorch (CPU/CUDA). See the project root `README.md`.
- **cuStateVec** (optional): NVIDIA cuQuantum Python, CuPy, and an NVIDIA GPU.  
  - Install for CUDA 12: `pip install --no-cache-dir cuquantum-python-cu12` (pulls CuPy etc.). Use `cuquantum-python-cu11` for CUDA 11.  
  - Without a GPU or cuQuantum, you can run FastSim only (`--no-custatevec` or `run_fastsim.py` directly).

## Install and run

From the project root:

```bash
# 1. Generate circuit and run both FastSim and cuStateVec (default 12 qubits, 500 gates)
python benchmarks/bench_statevector_vs_custatevec.py --num-qubits 12 --num-gates 500

# 2. Use an existing circuit spec JSON
python benchmarks/bench_statevector_vs_custatevec.py --spec path/to/circuit_spec.json

# 3. FastSim only (no cuQuantum)
python benchmarks/bench_statevector_vs_custatevec.py --num-qubits 10 --num-gates 200 --no-custatevec

# 4. Save generated circuit, more warmup/repeat, output JSON
python benchmarks/bench_statevector_vs_custatevec.py --num-qubits 14 --num-gates 1000 \
  --save-spec benchmarks/my_circuit.json --warmup 5 --repeat 30 --output-json results.json
```

Run a single backend:

```bash
# FastSim only (generate or provide a circuit spec first)
python benchmarks/run_fastsim.py benchmarks/my_circuit.json --device cuda --warmup 5 --repeat 20

# cuStateVec only (requires cuQuantum + CuPy)
python benchmarks/run_custatevec.py benchmarks/my_circuit.json --warmup 5 --repeat 20
```

Generate a circuit spec without running the benchmark:

```bash
python benchmarks/circuit_generator.py --num-qubits 10 --num-gates 100 --seed 42 --output my_spec.json
```

## Verifying result consistency

To check that both simulators produce the same final state (after accounting for bit ordering):

```bash
python benchmarks/bench_statevector_vs_custatevec.py --spec path/to/spec.json --verify
```

- FastSim uses **big-endian** state indexing (qubit 0 = MSB); cuStateVec uses **little-endian** (qubit 0 = LSB). The script converts one to the other and reports fidelity and max absolute difference.
- Verification runs only when both backends are used and `num_qubits <= --verify-max-qubits` (default 20). Use `--verify-atol` and `--verify-rtol` to tune tolerances.
- **Numerical precision**: Both backends use single-precision (complex64). For large gate counts (e.g. hundreds of gates) or many qubits, rounding can accumulate and verification may report FAIL even when the implementations match. For a reliable consistency check, use smaller circuits (e.g. `--num-gates 20` or `--num-qubits 8`).

### Ensuring every config gate is tested (`--cover-all-gates`)

To **guarantee** that each gate type in `gates_config.json` is exercised at least once (and that FastSim and cuStateVec agree on all of them), use cover-all-gates mode:

```bash
python benchmarks/bench_statevector_vs_custatevec.py --cover-all-gates --verify
```

This generates a circuit with **exactly one application of each gate** defined in the config (in config order), then runs both backends and verifies. The script prints the list of gate types tested (e.g. `X, Y, Z, H, CNOT, RX, RY, RZ, ZZ, CY, CZ, ...`). `--num-qubits` is forced to at least 3 (required for TOFFOLI).

## Workload and bit-order convention

- **Circuit spec**: Generated by `circuit_generator.py` or loaded from JSON: list of `{"gate_name": "...", "qubits": [...], "parameters": ...}`.
- **Default gate set**: H, X, Y, Z, CNOT, CZ, SWAP (non-parametric). Parametric gates (e.g. RX, RY, RZ) are supported when `parameters` are present in the spec.
- **Bit order**: cuStateVec uses **little-endian** state indices (qubit 0 = LSB). Both apply the same gates to the same qubit indices so the workload is comparable. For timing-only comparison, no conversion is needed.
- **Precision**: Both use complex64.

## Metrics

- **Primary**: Wall-clock time (seconds) for one full circuit forward. The script reports **mean ± std** after warmup.
- **Optional**: `--memory` reports FastSim peak GPU memory (MiB). cuStateVec memory stats are environment-dependent.

## Reproducibility

Run on the same machine and GPU and record:

- GPU model, driver version, CUDA version  
- PyTorch version, cuQuantum version (if used), Python version  

## Files

| File | Description |
|------|-------------|
| `circuit_generator.py` | Reproducible circuit generation (num_qubits, num_gates, seed, gate_set) |
| `run_fastsim.py` | FastSim single-backend run and timing (with GPU sync) |
| `run_custatevec.py` | cuStateVec single-backend run and timing (requires cuQuantum) |
| `verify_consistency.py` | Bit-order conversion and fidelity / tolerance check for final states |
| `bench_statevector_vs_custatevec.py` | Main entry: generate/load spec, run FastSim and cuStateVec, output comparison or JSON |
